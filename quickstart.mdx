---
title: "Set up CodeAtlas in under ten minutes"
description: "Install dependencies, configure environment variables, start the development servers, and analyze your first GitHub repository with CodeAtlas."
---

CodeAtlas requires a Supabase project for its database and auth layer, and a Google Gemini API key for AI-powered analysis. This guide walks you through every step — from cloning the repository to viewing your first architectural knowledge map.

<Steps>

<Step title="Meet the prerequisites">

Before you begin, make sure the following are in place:

- **Node.js 20 or later** — CodeAtlas uses the npm workspace protocol for its monorepo. Run `node -v` to confirm your version.
- **Supabase account** — You need a Supabase project to provide the PostgreSQL database, vector storage, and authentication layer. Create one at [supabase.com](https://supabase.com).
- **Google Gemini API key** — The analysis and security audit pipelines call Gemini models. Get a key from [Google AI Studio](https://aistudio.google.com/app/apikey).

<Note>
A GitHub account is only required for private repository ingestion. Public repositories are cloned over HTTPS without authentication.
</Note>

</Step>

<Step title="Clone the repository and install dependencies">

Clone CodeAtlas and install all workspace dependencies from the root:

<CodeGroup>

```bash npm
git clone https://github.com/Pragyat-Nikunj/CodeAtlas
cd CodeAtlas
npm install
```

```bash pnpm
git clone https://github.com/Pragyat-Nikunj/CodeAtlas
cd CodeAtlas
pnpm install
```

</CodeGroup>

The root `npm install` installs dependencies for all three workspaces — `frontend`, `backend`, and `packages/shared-schema` — in a single pass.

</Step>

<Step title="Configure environment variables">

CodeAtlas uses two separate `.env.local` files: one for the backend Express server, and one for the Next.js frontend.

**Backend — `backend/.env.local`**

```env
# Database
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_ANON_KEY=eyJhbGc...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGc...

# AI Services
GOOGLE_GENAI_API_KEY=AIzaSy...

# Server
NODE_ENV=development
PORT=5000

# Rate Limiting (optional — these are the defaults)
INGESTION_RATE_LIMIT_WINDOW_MS=900000
INGESTION_RATE_LIMIT_MAX=5
```

**Frontend — `frontend/.env.local`**

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...

# Backend API
NEXT_PUBLIC_API_URL=http://localhost:5000
```

<Warning>
`SUPABASE_SERVICE_ROLE_KEY` has full database access and bypasses row-level security. Never expose it to the browser or commit it to version control.
</Warning>

Find your Supabase keys under **Project Settings → API** in the Supabase dashboard. The service role key is on the same page, beneath the anon key.

</Step>

<Step title="Run database migrations">

From the `backend` directory, apply the database schema migrations to your Supabase project:

```bash
cd backend
npm run migrate
```

This creates the four tables CodeAtlas relies on: `projects`, `ingestion_jobs`, `documentation_nodes`, and `security_findings`.

</Step>

<Step title="Start the development servers">

Return to the root and start both servers concurrently:

```bash
npm run dev
```

This command runs `dev:frontend` and `dev:backend` in parallel using `concurrently`.

| Service | URL |
|---|---|
| Frontend (Next.js) | http://localhost:3000 |
| Backend (Express) | http://localhost:5000 |

You can also start each server individually:

```bash
npm run dev:frontend   # Next.js only
npm run dev:backend    # Express only
```

</Step>

<Step title="Analyze your first repository">

You can submit a repository through the web UI at http://localhost:3000, or directly via the API.

To use the API, you need a bearer token from your Supabase session. With a token in hand, POST a repository URL to the `/api/projects` endpoint:

```bash
curl -X POST http://localhost:5000/api/projects \
  -H "Authorization: Bearer your_supabase_jwt" \
  -H "Content-Type: application/json" \
  -d '{
    "githubUrl": "https://github.com/facebook/react"
  }'
```

A successful response returns a `projectId` and a `jobId`:

```json
{
  "success": true,
  "data": {
    "projectId": "proj_123...",
    "jobId": "job_456...",
    "repository": "facebook/react"
  }
}
```

<Tip>
The ingestion endpoint is rate-limited to **5 requests per 15 minutes** per IP. This is configurable via `INGESTION_RATE_LIMIT_WINDOW_MS` and `INGESTION_RATE_LIMIT_MAX` in your backend environment.
</Tip>

</Step>

<Step title="Track the job and view results">

Poll the job endpoint using the `jobId` from the previous step to monitor pipeline progress:

```bash
curl http://localhost:5000/api/jobs/job_456...
```

The response includes a `status` field and a `progress` percentage (0–100):

```json
{
  "success": true,
  "data": {
    "status": "ANALYZING",
    "progress": 45,
    "projectId": "proj_123...",
    "error": null
  }
}
```

Jobs move through these states: `PENDING → CLONING → ANALYZING → COMPLETED`. Once the status is `COMPLETED`, fetch the full results:

```bash
# Architectural documentation tree
curl http://localhost:5000/api/projects/proj_123.../nodes

# Security audit findings
curl http://localhost:5000/api/projects/proj_123.../security
```

In the web UI, the knowledge map becomes interactive as soon as the job completes — no refresh required.

</Step>

</Steps>
