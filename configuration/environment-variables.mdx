CodeAtlas uses two separate `.env.local` files to configure the backend Express server and the Next.js frontend. This page lists every variable, its purpose, and whether it is required or optional. Both files are excluded from version control by default; never commit either file to a public repository.

## Backend — `backend/.env.local`

The backend reads these variables at startup. `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are validated immediately on import; the server will throw and refuse to start if either is missing.

### Supabase

<ParamField path="SUPABASE_URL" type="string" required>
  The base URL of your Supabase project, in the form `https://<project-ref>.supabase.co`. Find this under **Project Settings → API** in the Supabase dashboard.
</ParamField>

<ParamField path="SUPABASE_ANON_KEY" type="string" required>
  The anon/public key for your Supabase project. This key is safe to use in server-side code and is used to verify JWT tokens issued by Supabase Auth.
</ParamField>

<ParamField path="SUPABASE_SERVICE_ROLE_KEY" type="string" required>
  The service role key for your Supabase project. This key bypasses row-level security and has full database access. The backend uses it to write ingestion results to the database.
</ParamField>

<Warning>
  `SUPABASE_SERVICE_ROLE_KEY` grants unrestricted access to your Supabase database. Never expose it to the browser, include it in frontend builds, or commit it to version control.
</Warning>

### AI services

<ParamField path="GOOGLE_GENAI_API_KEY" type="string" required>
  Your Google Gemini API key. The analysis pipeline calls Gemini models for structural analysis, security auditing, and embedding generation. Get a key from [Google AI Studio](https://aistudio.google.com/app/apikey).
</ParamField>

### Server

<ParamField path="NODE_ENV" type="string" default="development">
  The application environment. Accepted values: `development`, `production`, `test`. In `production`, stack traces are omitted from error responses.
</ParamField>

<ParamField path="PORT" type="number" default="5000">
  The port the Express server listens on. If not set, the server defaults to port `5000`.
</ParamField>

<Note>
  When running both servers locally with `npm run dev`, the backend defaults to port `5000` and the frontend to port `3000`. Set `PORT` only if you need a different port.
</Note>

<ParamField path="LOG_LEVEL" type="string" default="info">
  The Winston log level. Controls which messages are written to the console. Accepted values: `error`, `warn`, `info`, `debug`. Use `debug` for verbose output during development.
</ParamField>

### Rate limiting

<ParamField path="INGESTION_RATE_LIMIT_WINDOW_MS" type="number" default="900000">
  The time window for the ingestion rate limiter, in milliseconds. The default value of `900000` equals 15 minutes. Requests exceeding `INGESTION_RATE_LIMIT_MAX` within this window receive a `429` response.
</ParamField>

<ParamField path="INGESTION_RATE_LIMIT_MAX" type="number" default="5">
  The maximum number of repository ingestion requests allowed per IP address within the window defined by `INGESTION_RATE_LIMIT_WINDOW_MS`. The default is `5` requests per 15 minutes.
</ParamField>

<Note>
  The global rate limiter (100 requests per minute) is hardcoded and cannot be changed via environment variables. Only the ingestion-specific limit is configurable.
</Note>

### Complete backend example

```env backend/.env.local
# Supabase
SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxxx.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# AI
GOOGLE_GENAI_API_KEY=AIzaSy...

# Server
NODE_ENV=development
PORT=5000
LOG_LEVEL=info

# Rate limiting (optional — these are the defaults)
INGESTION_RATE_LIMIT_WINDOW_MS=900000
INGESTION_RATE_LIMIT_MAX=5
```

## Frontend — `frontend/.env.local`

The Next.js frontend uses three variables. All are prefixed with `NEXT_PUBLIC_`, which means they are bundled into the client-side JavaScript and visible in the browser. Do not place secret keys here.

### Supabase

<ParamField path="NEXT_PUBLIC_SUPABASE_URL" type="string" required>
  The same Supabase project URL used by the backend (`https://<project-ref>.supabase.co`). The frontend uses this to initialize the Supabase browser client for authentication.
</ParamField>

<ParamField path="NEXT_PUBLIC_SUPABASE_ANON_KEY" type="string" required>
  The anon/public key for your Supabase project — the same value as the backend's `SUPABASE_ANON_KEY`. This is safe to expose to the browser and is used by the Supabase Auth client to sign users in and out.
</ParamField>

### API

<ParamField path="NEXT_PUBLIC_API_URL" type="string" required>
  The base URL of the CodeAtlas backend API. In local development this is `http://localhost:5000`. In production, set this to your deployed backend URL (for example, `https://api.yourapp.com`).
</ParamField>

### Complete frontend example

```env frontend/.env.local
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Backend API
NEXT_PUBLIC_API_URL=http://localhost:5000
```

## Where to find your Supabase keys

<Steps>
  <Step title="Open your Supabase project">
    Go to [supabase.com](https://supabase.com) and open the project you created for CodeAtlas.
  </Step>
  <Step title="Navigate to API settings">
    In the left sidebar, click **Project Settings**, then select **API**.
  </Step>
  <Step title="Copy the keys">
    - **Project URL** → `SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_URL`
    - **anon / public** key → `SUPABASE_ANON_KEY` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`
    - **service_role** key → `SUPABASE_SERVICE_ROLE_KEY` (backend only)
  </Step>
</Steps>
